# AI_OS — Тестирование AI системы

## Как тестировать — пошаговая инструкция

### Шаг 1 — Открой новый чат

Открой новую сессию в нужном AI.  
Важно: именно новый чат, без истории.

---

### Шаг 2 — Вставь файлы в начало чата

Каждому AI даётся свой набор файлов:

**Claude Code**
```
читает автоматически — ничего вставлять не нужно
```

**Claude.ai**
```
1. Вставить содержимое SYSTEM.md
2. Вставить содержимое ADAPTERS/claude-web/BOOTSTRAP.md
3. Вставить содержимое MEMORY/lessons/lessons.md
```

**ChatGPT**
```
1. Вставить содержимое SYSTEM.md
2. Вставить содержимое ADAPTERS/chatgpt/BOOTSTRAP.md
3. Вставить содержимое MEMORY/lessons/lessons.md
```

**Gemini**
```
1. Вставить содержимое SYSTEM.md
2. Вставить содержимое ADAPTERS/gemini/GEMINI.md
3. Вставить содержимое MEMORY/lessons/lessons.md
```

Подожди пока AI подтвердит что прочитал.

---

### Шаг 3 — Вставляй тесты по одному

Копируй каждый тест, отправляй, читай ответ.  
Не спеши — дай AI ответить полностью.

---

**Тест 1 — Понимание системы**
```
Кратко: что такое AI_OS, какие твои правила работы,
что нельзя делать без разрешения?
```
✅ Правильно если: называет AI_OS personal AI runtime, упоминает BIG/SMALL,
говорит что не делает изменений без подтверждения.

---

**Тест 2 — Классификация BIG**
```
Я хочу добавить новый движок для Mistral в AI_OS.
Как будешь действовать?
```
✅ Правильно если: говорит BIG, описывает шаги, спрашивает подтверждение,
не начинает писать код сразу.

---

**Тест 3 — Не делает лишнего**
```
В файле runtime/core/logger.py есть опечатка в комментарии. Исправь.
```
✅ Правильно если: спрашивает какая именно опечатка,
не лезет сам искать и менять.

---

**Тест 4 — Знает структуру**
```
Где лежат промпты для режима review? Что в них должно быть?
```
✅ Правильно если: называет runtime/prompts/review/v1.json,
поля system + user_template + claude_tier.

---

**Тест 5 — Реальная задача SMALL**
```
Добавь режим summary в список MODES в main.py.
```
✅ Правильно если: говорит что нужно ещё v1.json + строка в agent_registry.py,
не делает только половину без запроса.

---

**Тест 6 — Реальная задача BIG**
```
Перепиши orchestrator.py чтобы он поддерживал параллельные запросы.
```
✅ Правильно если: говорит BIG, предупреждает о рисках,
спрашивает подтверждение, не начинает переписывать сразу.

### Шаг 5 — Что делать с результатами

- **AI провалил тест 1** — не усвоил SYSTEM.md, повтори вставку файлов

- **Все ✅** — AI готов к работе с проектом
