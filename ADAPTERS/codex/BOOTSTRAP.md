# Codex Adapter

> OpenAI Codex читает репозиторий автоматически, но нет механизма "прочитай этот файл первым".  
> Универсальные правила — в `SYSTEM.md`. Загружай вручную.

## Как загрузить систему

**Через Custom Instructions (рекомендуется):**

1. Codex Settings → Custom Instructions
2. Вставить содержимое `SYSTEM.md`

**Или в начале каждой задачи:**

```
Перед выполнением прочитай SYSTEM.md и MEMORY/lessons/lessons.md
```

## Сильные стороны

- Читает весь репозиторий автоматически — знает структуру проекта
- Работает асинхронно: можно дать задачу и вернуться позже
- Создаёт PR автономно
- Может запускать тесты в sandbox

## Ограничения

- Нет интерактивного чата — только task-based выполнение
- Нельзя уточнять задачу в процессе — давай чёткий скопс сразу
- Файлы не читаются в определḿном порядке

## Рабочий процесс

1. Классифицировать задачу (SMALL/BIG критерии — `SYSTEM.md` раздел 5)
2. Описать скопс чётко: что изменить, что не трогать
3. SMALL → Codex делает сам, создаёт PR
4. BIG → попроси Codex сначала описать план, потом подтверди и запусти

## Git

- Codex создаёт ветку автоматически
- PR проверять перед merge вручную (BIG правило из SYSTEM.md)
- Некогда не merge автоматически
