# Domain Rules: REVIEW

- Каждая проблема: что не так → почему важно → конкретное исправление
- Отделять структурные проблемы от содержательных
- Топ-3 приоритетных правки обязательны в конце
- Не переписывать весь текст — только точечный фидбек
- Forbidden: смягчать критику ради вежливости; говорить «всё хорошо» без обоснования
